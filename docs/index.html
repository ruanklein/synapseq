Introduction to SynapSeq

This guide will help you get started with SynapSeq, a text-based tool for generating tones to support meditation, relaxation, and altered states of consciousness.

SynapSeq is built on the philosophy of doing one thing well. Providing a concise, human-readable syntax that lets you focus on what you want to achieve, not on how to express it.

One way to do it

Single, clear approach for each task

Clarity over cleverness

Readable and straightforward syntax

Intention over syntax

Focus on what you want to achieve

Less options, more focus

Essential features only

Brainwave Entrainment

Brainwave entrainment is a technique that uses rhythmic stimuli - such as pulsing sounds or flashing lights - to encourage the brain to synchronize its brainwaves with the frequency of the external stimulus.

Types of Entrainment

Binaural Beats

When two slightly different frequencies are played in each ear, the brain perceives a third "beat" frequency equal to the difference between the two tones.

Important: Binaural beats require headphones to be effective.

Example: 200Hz in left ear + 210Hz in right ear = 10Hz perceived beat

Monaural Beats

Two binaural beats with mirrored frequency offsets are played in opposite channels, creating a rhythmic pulse effective with or without headphones.

Isochronic Tones

Single tones that are turned on and off at regular intervals. The brain responds to this rhythmic stimulation and resonates with the frequency.

Can be effective with or without headphones.

Frequency Bands

Delta 0.5-4 Hz

Deep sleep, healing, deep meditation

Theta 4-8 Hz

Dreaming sleep, meditation, creativity

Alpha 8-13 Hz

Relaxed alertness, calm, learning

Beta 13-30 Hz

Active thinking, focus, alertness

Gamma 30+ Hz

Higher mental activity, peak concentration

Understanding the Syntax

SynapSeq uses a specific syntax to create sequences. The syntax is based on the following elements:

  • tone - A single frequency (carrier frequency)
  • noise - A random signal for background sound
  • background - A user-defined sound played in the background
  • waveform - The shape of the tone
  • silence - A period with no sound
  • track - A directive used inside presets that inherit from templates

Tone Element

The fundamental building block for creating brainwave entrainment sequences.

Syntax
tone [carrier frequency] [type] [frequency offset] amplitude [amplitude value]

Examples

Binaural Beat 
tone 400 binaural 10 amplitude 10
Monaural Beat 
tone 300 monaural 10 amplitude 10
Isochronic Tone 
tone 200 isochronic 10 amplitude 10
Carrier Only (No Method) 
tone 100 amplitude 10

Noise Element

Add background ambiance with different types of noise.

Syntax
noise [type] amplitude [amplitude value]

Examples

White Noise 
noise white amplitude 5
Pink Noise 
noise pink amplitude 15
Brown Noise 
noise brown amplitude 30

Background Element

Use custom audio files as background with optional effects.

Basic Syntax
background amplitude [amplitude value]

Effects

Spin Effect

Creates a sensation of circular sound movement between your ears.

Spin Example 
background spin 300 rate 7.5 intensity 30 amplitude 50

Pulse Effect

Creates a rhythmic pulsing sensation by modulating the amplitude.

Pulse Example 
background pulse 7.5 intensity 40 amplitude 50
Each preset can have only one background.

Waveform Element

Define the shape of tones and background effects.

sine Smooth, pure tone (default)
square Sharp, digital sound
triangle Softer than square
sawtooth Bright, buzzing tone

Examples

Square Waveform Tone 
waveform square tone 400 isochronic 10 amplitude 2.5
Triangle Background with Spin 
waveform triangle background spin 500 rate 5.5 intensity 30 amplitude 10

Presets

Presets are reusable configurations that define a specific state or sound combination.

Syntax
[preset name]
[elements] ...

Example

Alpha Preset 
alpha
  noise brown amplitude 40
  tone 100 binaural 8 amplitude 15

Templating Presets

Create reusable base templates that can be inherited and modified by other presets.

Declaring a Template

Template Declaration 
base-focus as template
  noise white amplitude 25
  tone 180 binaural 8 amplitude 10

Creating Derived Presets

Derived Presets 
focus-phase1 from base-focus
  track 1 amplitude 20
  track 2 tone 200

focus-phase2 from base-focus
  track 1 amplitude 30
  track 2 tone 220
Note: Template presets cannot be used directly in the timeline. They serve as a base for derived presets.

Accepted Track Parameters

  • tone <value> - carrier frequency (Hz)
  • binaural <value> / monaural <value> / isochronic <value> - beat frequency (Hz)
  • amplitude <value> - loudness (0–100%)
  • intensity <value> - effect intensity (for spin/pulse)
  • spin <value> - spin width effect for background track
  • pulse <value> - pulse effect for background track

Timeline

The timeline controls when presets play and how they transition.

Syntax
hh:mm:ss [preset name]

Example

Timeline Example 
00:00:00 silence
00:00:15 alpha1
00:01:00 alpha1
00:02:00 alpha2
00:03:00 silence

Fade In/Out

Use the silence keyword to create smooth fade-ins and fade-outs.

Transitions Between Different Types

In SynapSeq V3+, transitions between presets are smooth for elements of the same type. For different types (e.g., binaural to isochronic), you must explicitly define the transition.

Method 1: Using Silence

Fade Between Different Types 
00:00:00 silence
00:00:15 ps1
00:02:00 ps1
# Fade-out
00:02:30 silence
# Fade-in
00:03:00 ps2

Method 2: Manual Amplitude Control

Amplitude Crossfade 
alpha1
  tone 300 binaural 10 amplitude 10
  tone 300 isochronic 10 amplitude 0

alpha2
  tone 300 binaural 10 amplitude 0
  tone 300 isochronic 10 amplitude 10

Transitions

Control how parameters change between presets over time.

Syntax
hh:mm:ss [preset name] [transition type]

Transition Types

steady Steady (Default)

Constant rate of change throughout the entire transition period.

Progress: 0% —— 25% —— 50% —— 75% —— 100% (uniform change)

Characteristics:

  • Uniform progression from start to finish
  • Neutral, mechanical feel
  • Predictable and consistent

Best for: Testing, debugging, predictable linear changes

Provides a steady, predictable stimulus that works well as a baseline reference, though it may be more noticeable to the brain than natural transitions.

ease-out Ease-Out (Logarithmic)

Starts with rapid change and gradually slows down, creating a smooth landing.

Progress: 0% —— 60% —— 80% —— 90% —— 100% (fast start, gentle ending)

Characteristics:

  • Most change happens early in the transition
  • Gradually decelerates as it approaches the target
  • Like a car smoothly braking to a stop

Best for: High to low frequencies (beta → theta), relaxation, meditation entry

Mimics the natural process of falling asleep or entering relaxation. The rapid initial change captures the brain's attention and begins the shift, while the gentle ending allows the nervous system to stabilize comfortably in the new state without resistance.

ease-in Ease-In (Exponential)

Starts gently and accelerates toward the end, creating a smooth departure.

Progress: 0% —— 10% —— 20% —— 40% —— 100% (gentle start, fast ending)

Characteristics:

  • Starts slowly and gradually accelerates
  • Most change happens near the end
  • Like a car smoothly accelerating from a stop

Best for: Low to high frequencies (theta → beta), awakening, activation

Mirrors the natural awakening process. The gentle start avoids shocking the nervous system when emerging from deep states, while the accelerating finish firmly establishes the new alert state without leaving residual drowsiness.

smooth Smooth (Sigmoid)

Most natural feeling with gentle starts and endings, faster change in the middle.

Progress: 0% —— 20% —— 50% —— 80% —— 100% (slow → fast → slow, S-shaped)

Characteristics:

  • Starts slowly (ease-in)
  • Accelerates in the middle
  • Ends slowly (ease-out)
  • S-shaped curve, most natural and organic

Best for: General-purpose transitions, maximum comfort, therapeutic sessions

Provides the most comfortable and natural-feeling transition. The gentle start and end minimize perception of change, while the middle acceleration ensures the transition completes smoothly. This approach mimics natural processes and feels organic to the nervous system, making it ideal for therapeutic and meditative applications.

Example

Using Transitions 
00:00:00 silence
00:00:30 awake
# Entering meditation - use EASE-OUT
00:01:00 awake ease-out
00:06:00 deep
# Awakening - use EASE-IN
00:10:30 deep ease-in
00:15:00 awake
00:16:00 silence

Comments

Comments help document your sequences and are ignored during processing.

Syntax
# [comment]

Regular Comments

Single # Comment 
# This comment is ignored
alpha
  tone 440 binaural 8 amplitude 5

Printed Comments

Use double ## to print comments in the output.

Double ## Comment 
## This comment will be printed in the output
alpha
  tone 440 binaural 8 amplitude 5
Comments must occupy an entire line. Inline comments will cause syntax errors.

Global Options

Global options are set at the top of your file and start with @.

@background

Set a background audio file to loop throughout your sequence.

Examples 
@background /path/to/background.wav
@background ~/Downloads/rain.wav
@background https://example.com/sounds/rain.wav
Requirements:
  • Format: .wav files (8-bit/16-bit/24-bit)
  • Channels: 2 (stereo)
  • Sample rate must match sequence sample rate
  • Maximum file size: 10 MB

@gainlevel

Set the attenuation level of the background sound.

Example 
@gainlevel medium
off No attenuation (0dB) - full volume
high Light attenuation (-3dB)
medium Moderate attenuation (-9dB)
low Strong attenuation (-18dB)

@volume

Set the overall output volume (0-100).

Example 
@volume 75

Default: 100

@samplerate

Set the sample rate of the output.

Example 
@samplerate 44100

Default: 44100

@presetlist

Import presets from external files for reusability.

Examples 
@presetlist /path/to/my-presets.spsq
@presetlist ~/sequences/relaxation-presets.spsq
@presetlist https://example.com/presets/focus-presets.spsq
Only preset definitions are imported. Timeline sections and global options in preset files will trigger errors.

Command Line

Syntax
synapseq [options] [input file] [output file]

Basic Usage

Generate WAV from sequence 
synapseq sample-binaural.spsq sample-binaural.wav

Using stdin

You can use - to open sequence from stdin:

Pipe from stdin 
cat example.spsq | synapseq - output.wav

Using URLs

You can also use HTTP or HTTPS URLs to load sequences directly from the web:

Load from URL 
synapseq https://example.com/sequences/my-sequence.spsq output.wav

RAW Output Format

On *nix systems, you can play a sequence in RAW format using other audio tools:

Pipe to play (sox) 
synapseq sample-binaural.spsq - | play -t raw -r 44100 -e signed-integer -b 16 -c 2 -

RAW Audio Parameters

If you want to use another tool to process the output, keep in mind that the audio is emitted in RAW format with the following parameters:

  • Type: RAW
  • Sample Rate: 44100 Hz (default, configurable with @samplerate)
  • Encoding: Signed Integer
  • Bit Depth: 16
  • Channels: 2 (stereo)

Hub Commands

SynapSeq Hub is a centralized repository of community-contributed sequences. You can browse, search, download, and generate audio files directly from the Hub.

Explore sequences at SynapSeq Hub Repository.

Important: Before using any Hub command, you must first run -hub-update to fetch the latest manifest.

Privacy & Analytics

When you download sequences using -hub-get, -hub-info, or -hub-download, SynapSeq sends anonymous usage statistics.

What is collected:

  • Sequence ID (which sequence was downloaded)
  • SynapSeq version
  • Platform (Windows, macOS, Linux)
  • Architecture (amd64, arm64)
  • Action performed (get, info, or download)

What is NOT collected:

  • No personal information (names, emails, usernames)
  • No IP addresses or location data
  • No file paths or local directory structure
  • No browsing history or system information

The tracking is done asynchronously and never blocks or delays your downloads. All tracking logic is open source and can be fully inspected.

Compiling without Hub functionality: You can compile a Hub-disabled version using the nohub build tag. See the Compilation Guide for instructions.

-hub-update

Updates the local index of available sequences from the Hub.

Update Hub manifest 
synapseq -hub-update

-hub-list

Lists all available sequences in the Hub.

List all sequences 
synapseq -hub-list

-hub-search

Searches for sequences in the Hub by keyword. The search is case-insensitive and matches against the sequence ID, name, author, and category.

Search for sequences 
synapseq -hub-search focus

-hub-download

Downloads a sequence and all its dependencies to a local directory.

Download sequence 
synapseq -hub-download synapseq.samples.genesis ./downloads

If no target directory is specified, the current directory (.) is used.

-hub-get

Generates audio output directly from a Hub sequence without downloading it first.

Basic Usage (WAV output)

Generate WAV from Hub 
synapseq -hub-get synapseq.samples.genesis output.wav

Stream to stdout

Stream RAW audio 
synapseq -hub-get synapseq.samples.genesis -

Play in real-time

Stream and play 
synapseq -hub-get synapseq.samples.genesis -play

Convert to MP3

Generate MP3 from Hub 
synapseq -hub-get synapseq.samples.genesis -mp3

-hub-clean

Cleans the local Hub cache, removing all cached manifest data and downloaded temporary files.

Clean Hub cache 
synapseq -hub-clean

Note: After cleaning the cache, you'll need to run -hub-update again.

Additional Options

Extract

Extracts the original sequence and metadata embedded in a WAV or MP3 file.

Extract from WAV

Extract from WAV 
synapseq -extract input.wav output.spsq

Extract from MP3

To extract sequences from MP3 files, you must use the -mp3 flag and have ffprobe installed.

Extract from MP3 
synapseq -extract -mp3 input.mp3 output.spsq

Playing Mode

Plays a sequence in real-time using ffplay. The audio is streamed directly without creating intermediate files.

Requirements: This feature requires ffplay (part of the FFmpeg suite) to be installed.
Play sequence 
synapseq -play sample-binaural.spsq
Play from URL 
synapseq -play https://example.com/sequences/my-sequence.spsq

Export Options

SynapSeq can export sequences to different audio formats using ffmpeg.

Requirements: MP3 export feature requires ffmpeg to be installed.

MP3 Export

Converts a sequence to MP3 format. By default, creates an output file with the same name as the input but with .mp3 extension.

Auto-generate filename 
synapseq -mp3 sample.spsq
Custom output filename 
synapseq -mp3 sample.spsq output.mp3

MP3 Audio Quality

  • Codec: libmp3lame
  • Bitrate: 320 kbps (high quality)
  • Sample rate: Matches the sequence (default 44100 Hz)
  • Channels: Stereo (2 channels)

WAV Export

Generates standard WAV (uncompressed) audio files. This is the default export format when no format flag is specified.

Export to WAV 
synapseq sample.spsq output.wav

WAV Audio Quality

  • Format: Uncompressed PCM
  • Bit depth: 16-bit
  • Sample rate: Matches the sequence (default 44100 Hz)
  • Channels: Stereo (2 channels)

Structured Formats

SynapSeq supports reading sequences from structured data formats (JSON, XML, YAML) as an alternative to the native .spsq format. These formats use a common schema that represents all sequence components.

Converting to SPSQ

Use the -convert flag to convert a structured format (JSON, XML, YAML) back into .spsq format.

Convert JSON to SPSQ 
synapseq -convert -json input.json output.spsq
Convert XML to SPSQ 
synapseq -convert -xml input.xml output.spsq
Convert YAML to SPSQ 
synapseq -convert -yaml input.yaml output.spsq

File Structure

All structured formats follow this schema:

Common Schema Structure 
Root object
├── description: array of strings (converted from lines starting with "##" in .spsq)
├── options
│   ├── samplerate: integer (e.g., 44100)
│   ├── volume: integer 0–100
│   ├── background: string (optional) — path to background audio file (WAV)
│   │                Can be a local file path, home directory path (~), or HTTP/HTTPS URL
│   └── gainlevel: string (optional) — one of: low, medium, high, veryhigh (default: veryhigh)
└── sequence: array of entries
    └── entry
        ├── time: integer milliseconds (first must be 0, strictly increasing)
        ├── transition: steady | ease-out | ease-in | smooth
        └── track
            ├── tones: array of tone objects
            │   └── tone
            │       ├── mode: binaural | monaural | isochronic | pure
            │       ├── carrier: Hz (float)
            │       ├── resonance: Hz (float) — for beats
            │       ├── amplitude: 0–100 (float)
            │       └── waveform: sine | square | triangle | sawtooth
            ├── noises: array of noise objects
            │   └── noise
            │       ├── mode: white | pink | brown
            │       └── amplitude: 0–100 (float)
            └── background: object (optional) — background audio modulation
                ├── amplitude: 0–100 (float)
                ├── waveform: sine | square | triangle | sawtooth
                └── effect: object (optional)
                    ├── intensity: 0–100 (float) — effect depth
                    ├── spin: object (optional) — spatial rotation effect
                    │   ├── width: Hz (float) — rotation width
                    │   └── rate: Hz (float) — rotation speed
                    └── pulse: object (optional) — rhythmic pulsing effect
                        └── resonance: Hz (float) — pulse frequency

-ffmpeg-path

Specifies a custom path to the ffmpeg executable.

Custom ffmpeg path 
synapseq -mp3 -ffmpeg-path /usr/local/bin/ffmpeg sample.spsq output.mp3

-ffplay-path

Specifies a custom path to the ffplay executable.

Custom ffplay path 
synapseq -play -ffplay-path /usr/local/bin/ffplay sample.spsq

-ffprobe-path

Specifies a custom path to the ffprobe executable. Only needed when extracting sequences from MP3 files.

Custom ffprobe path 
synapseq -extract -mp3 -ffprobe-path /usr/local/bin/ffprobe input.mp3 output.spsq

-help

Show the help and exit.

-unsafe-no-metadata

Prevents embedding the original SPSQ sequence and metadata into the generated WAV or MP3 file.

Generate without metadata 
synapseq -unsafe-no-metadata sample.spsq output.wav

-quiet

Quiet mode. Used to hide terminal output. Errors and comments will be displayed.

-test

Test mode. Used to validate the syntax of a .spsq file or structured formats (JSON, XML, YAML) without generating any output file.

Test sequence syntax 
synapseq -test sample-binaural.spsq

-version

Show the version.

-install-file-association (Windows only)

Installs Windows file association and context menu integration for SynapSeq.

Features:

File Association:

Compiling SynapSeq

First, clone the repository:

Terminal 
git clone https://github.com/ruanklein/synapseq.git
cd synapseq

SynapSeq can be compiled using the provided Makefile.

For POSIX systems (macOS/Linux)

Simply run:

Terminal 
make

This will automatically compile SynapSeq for your current operating system and architecture, creating a binary in the bin/ directory.

For Windows

Windows users should use the platform-specific targets to ensure proper .exe extension, application icon, and Windows-specific command-line options:

Git Bash 
make build-windows-amd64    # Windows 64-bit (Intel/AMD) - Recommended
make build-windows-arm64    # Windows 64-bit (ARM)
Do not use make build on Windows, as it will create a binary without the .exe extension, missing the application icon and Windows-specific features.

For cross-compilation to other platforms

You can build for different platforms and architectures:

Terminal 
# Linux
make build-linux-amd64      # Linux 64-bit (Intel/AMD)
make build-linux-arm64      # Linux 64-bit (ARM)

# macOS
make build-macos            # macOS ARM64 (Apple Silicon)
For Windows builds: The Makefile automatically generates Windows resource files (including application icon and metadata) when building for Windows. This requires the goversioninfo tool, which will be automatically downloaded during the build process.

For WebAssembly (Browser)

You can compile SynapSeq to WebAssembly for use in web browsers:

Terminal 
make build-wasm

This will generate the WebAssembly files in the wasm/ directory:

  • synapseq.wasm - The WebAssembly binary
  • wasm_exec.js - Go WASM runtime (copied from Go installation)

macOS/Linux

Compiling without Hub Support

If you prefer to use SynapSeq without any Hub features (including analytics tracking), you can compile a Hub-disabled version using the nohub build tag. This completely removes all Hub-related code from the binary, resulting in a smaller executable with no network connections to the SynapSeq Hub infrastructure.

To compile without Hub support:

Terminal 
# Build for POSIX system (macOS/Linux)
make build-nohub

# Or build for specific platforms and architectures
make build-windows-nohub-amd64    # Windows 64-bit (Intel/AMD)
make build-windows-nohub-arm64    # Windows 64-bit (ARM)
make build-linux-nohub-amd64      # Linux 64-bit (Intel/AMD)
make build-linux-nohub-arm64      # Linux 64-bit (ARM)
make build-macos-nohub            # macOS ARM64 (Apple Silicon)

What's different in the Hub-disabled build:

  • All -hub-* commands will return an error message
  • No network connections to the Hub infrastructure
  • No analytics or tracking of any kind
  • Smaller binary size (Hub code is excluded)
  • All other SynapSeq features work normally

Installing the Hub-disabled binary:

After compilation, you can install it system-wide:

macOS/Linux 
sudo make install-nohub
Windows (Git Bash as Administrator) 
mkdir -p "/c/Program Files/SynapSeq"
cp bin/synapseq-windows-amd64-nohub.exe "/c/Program Files/SynapSeq/synapseq-nohub.exe"

Programming

SynapSeq provides libraries for integrating brainwave entrainment into your applications. Generate and play sequences programmatically in the browser or build custom tools in Go.

SynapSeq.js - WebAssembly Library

A JavaScript wrapper library for SynapSeq WASM, providing an elegant object-oriented API for generating and playing SynapSeq sequences directly in the browser.

Features

  • Real-time streaming audio generation
  • Generate binaural/monaural/isochronic tones from SPSQ sequences
  • Support for both text (SPSQ) and JSON formats
  • WebAssembly-powered for high performance
  • AudioWorklet-based streaming for low-latency playback
  • Integrated Web Worker for non-blocking audio generation
  • Single-file library with embedded worker
  • Support for local and remote WASM files
  • Full JSDoc documentation
  • Promise-based async operations
  • Built-in audio playback controls

Basic Usage 
// Create a new SynapSeq instance
const synapseq = new SynapSeq();

// Load a sequence in text format (SPSQ)
const spsqCode = `

  tone 250 isochronic 10 amplitude 15

# Timeline
00:00:00 silence
00:00:15 alpha
00:04:45 alpha
00:05:00 silence
`;

async function play() {
  // Load from string (text format is default)
  await synapseq.load(spsqCode);
  await synapseq.play();

  // Or load JSON format
  // await synapseq.load(jsonString, "json");

  // Or load from File object (format auto-detected)
  // const fileInput = document.getElementById("fileInput");
  // await synapseq.load(fileInput.files[0]);
}

play();

Using Custom Paths

You can specify custom paths for WASM files. By default, the library looks for files in the same directory. You can customize these paths:

Custom Paths 
// Local custom paths
const synapseq = new SynapSeq({
  wasmPath: "./dist/synapseq.wasm",
  wasmExecPath: "./dist/wasm_exec.js",
});

// Using SynapSeq CDN
const synapseq = new SynapSeq({
  wasmPath: "https://synapseq.org/wasm/synapseq.wasm",
  wasmExecPath: "https://synapseq.org/wasm/wasm_exec.js",
});

// Mixed (local lib, CDN WASM)
const synapseq = new SynapSeq({
  wasmPath: "https://synapseq.org/wasm/synapseq.wasm",
  // wasmExecPath defaults to local 'wasm_exec.js'
});

API Reference

Constructor

new SynapSeq(options)

Creates a new SynapSeq instance and initializes the embedded Web Worker.

Parameters:

  • options (Object) - Optional configuration object
    • wasmPath (string) - Path or URL to the WASM file (default: 'synapseq.wasm')
    • wasmExecPath (string) - Path or URL to the wasm_exec.js file (default: 'wasm_exec.js')

Returns: SynapSeq instance

Methods

load(input, format)

Loads a sequence from a string or File object. Supports both text (SPSQ) and JSON formats.

Parameters:

  • input (string|File) - Sequence content as string or File object
  • format (string) - Optional format: 'text' (default) or 'json'

Returns: Promise<void>

Throws: Error if input is invalid

Load Examples 
// Load from SPSQ text string (default format)
const spsqCode = `
# Presets
alpha
  tone 250 isochronic 10

# Timeline
00:00:00 alpha
`;
await synapseq.load(spsqCode);
// or explicitly:
await synapseq.load(spsqCode, "text");

// Load from File object (.spsq file)
const fileInput = document.getElementById("fileInput");
await synapseq.load(fileInput.files[0], "text");

// Load from File object (.json file)
await synapseq.load(fileInput.files[0], "json");
play()

Plays the loaded sequence. Streams and generates audio in real-time.

Returns: Promise<void>

Throws: Error if no sequence is loaded

stop()

Stops the currently playing sequence and resets playback position.

Returns: void

getCurrentTime()

Gets the current playback position in seconds since playback started.

Returns: number - Current time in seconds (0 if not playing)

getSampleRate()

Gets the sample rate of the loaded sequence.

Returns: number - Sample rate in Hz

getState()

Gets the current playback state.

Returns: string - One of: 'idle', 'playing', 'stopped'

isLoaded()

Checks if a sequence is currently loaded.

Returns: boolean - True if a sequence is loaded

isReady()

Checks if the Web Worker is initialized and ready.

Returns: boolean - True if worker is ready

getVersion()

Gets the SynapSeq version from the WASM module.

Returns: Promise<string> - The version string

getBuildDate()

Gets the build date of the SynapSeq WASM module.

Returns: Promise<string> - The build date string

getHash()

Gets the hash of the SynapSeq WASM build.

Returns: Promise<string> - The hash string

destroy()

Cleans up resources and terminates the Web Worker.

Returns: void

Event Handlers

All event handlers are optional callback functions that can be assigned to handle different states.

onloaded

Called when a sequence is successfully loaded.

synapseq.onloaded = () => {
  console.log("Sequence loaded and ready to play");
};
ongenerating

Called when audio generation starts.

synapseq.ongenerating = () => {
  console.log("Generating audio...");
};
onplaying

Called when playback starts.

synapseq.onplaying = () => {
  console.log("Now playing");
};
onstopped

Called when playback is stopped.

synapseq.onstopped = () => {
  console.log("Playback stopped");
};
onended

Called when playback ends naturally.

synapseq.onended = () => {
  console.log("Playback finished");
};
onerror

Called when an error occurs.

Parameters:

  • detail (Object) - Contains error property with the Error object 
synapseq.onerror = (detail) => {
  console.error("Error occurred:", detail.error);
};

Complete Example

Full Implementation 
// Create instance
const synapseq = new SynapSeq();

// Setup event handlers
synapseq.onloaded = () => {
  console.log("Sequence loaded!");
  document.getElementById("playBtn").disabled = false;
};

synapseq.ongenerating = () => {
  console.log("Generating audio...");
  document.getElementById("status").textContent = "Generating...";
};

synapseq.onplaying = () => {
  console.log("Playing");
  document.getElementById("status").textContent = "Playing";
  document.getElementById("playBtn").disabled = true;
  document.getElementById("stopBtn").disabled = false;
};

synapseq.onstopped = () => {
  console.log("Stopped");
  document.getElementById("status").textContent = "Stopped";
  document.getElementById("playBtn").disabled = false;
  document.getElementById("stopBtn").disabled = true;
};

synapseq.onended = () => {
  console.log("Finished");
  document.getElementById("status").textContent = "Finished";
  document.getElementById("playBtn").disabled = false;
  document.getElementById("stopBtn").disabled = true;
};

synapseq.onerror = (detail) => {
  console.error("Error:", detail.error);
  alert("Error: " + detail.error.message);
};

// Load and play
async function start() {
  const spsqCode = `
# Presets
alpha-one
  noise pink amplitude 30
  tone 250 isochronic 8 amplitude 15

alpha-two
  noise pink amplitude 25
  tone 250 isochronic 10 amplitude 15

# Timeline
00:00:00 silence
00:00:15 alpha-one
00:02:00 alpha-two
00:03:00 alpha-two
00:04:00 silence
  `;

  try {
    await synapseq.load(spsqCode);
    await synapseq.play();
  } catch (error) {
    console.error("Failed to start:", error);
  }
}

// Control functions
function stop() {
  synapseq.stop();
}

// Progress tracking
setInterval(() => {
  const current = synapseq.getCurrentTime();
  const sampleRate = synapseq.getSampleRate();
  if (current > 0) {
    document.getElementById("time").textContent = 
      `Time: ${current.toFixed(1)}s @ ${sampleRate}Hz`;
  }
}, 100);
