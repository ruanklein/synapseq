Introduction to SynapSeq

This guide will help you get started with SynapSeq, a text-based tool for generating tones to support meditation, relaxation, and altered states of consciousness.

SynapSeq is built on the philosophy of doing one thing well. Providing a concise, human-readable syntax that lets you focus on what you want to achieve, not on how to express it.

One way to do it

Single, clear approach for each task

Clarity over cleverness

Readable and straightforward syntax

Intention over syntax

Focus on what you want to achieve

Less options, more focus

Essential features only

Brainwave Entrainment

Brainwave entrainment is a technique that uses rhythmic stimuli - such as pulsing sounds or flashing lights - to encourage the brain to synchronize its brainwaves with the frequency of the external stimulus.

Types of Entrainment

Binaural Beats

When two slightly different frequencies are played in each ear, the brain perceives a third "beat" frequency equal to the difference between the two tones.

Important: Binaural beats require headphones to be effective.

Example: 200Hz in left ear + 210Hz in right ear = 10Hz perceived beat

Monaural Beats

Two binaural beats with mirrored frequency offsets are played in opposite channels, creating a rhythmic pulse effective with or without headphones.

Isochronic Tones

Single tones that are turned on and off at regular intervals. The brain responds to this rhythmic stimulation and resonates with the frequency.

Can be effective with or without headphones.

Frequency Bands

Delta 0.5-4 Hz

Deep sleep, healing, deep meditation

Theta 4-8 Hz

Dreaming sleep, meditation, creativity

Alpha 8-13 Hz

Relaxed alertness, calm, learning

Beta 13-30 Hz

Active thinking, focus, alertness

Gamma 30+ Hz

Higher mental activity, peak concentration

Understanding the Syntax

SynapSeq uses a specific syntax to create sequences. The syntax is based on the following elements:

  • tone - A single frequency (carrier frequency)
  • noise - A random signal for background sound
  • background - A user-defined sound played in the background
  • waveform - The shape of the tone
  • silence - A period with no sound
  • track - A directive used inside presets that inherit from templates

Tone Element

The fundamental building block for creating brainwave entrainment sequences.

Syntax
tone [carrier frequency] [type] [frequency offset] amplitude [amplitude value]

Examples

Binaural Beat 
tone 400 binaural 10 amplitude 10
Monaural Beat 
tone 300 monaural 10 amplitude 10
Isochronic Tone 
tone 200 isochronic 10 amplitude 10
Carrier Only (No Method) 
tone 100 amplitude 10

Noise Element

Add background ambiance with different types of noise.

Syntax
noise [type] amplitude [amplitude value]

Examples

White Noise 
noise white amplitude 5
Pink Noise 
noise pink amplitude 15
Brown Noise 
noise brown amplitude 30

Background Element

Use custom audio files as background with optional effects.

Basic Syntax
background amplitude [amplitude value]

Effects

Spin Effect

Creates a sensation of circular sound movement between your ears.

Spin Example 
background spin 300 rate 7.5 intensity 30 amplitude 50

Pulse Effect

Creates a rhythmic pulsing sensation by modulating the amplitude.

Pulse Example 
background pulse 7.5 intensity 40 amplitude 50
Each preset can have only one background.

Waveform Element

Define the shape of tones and background effects.

sine Smooth, pure tone (default)
square Sharp, digital sound
triangle Softer than square
sawtooth Bright, buzzing tone

Examples

Square Waveform Tone 
waveform square tone 400 isochronic 10 amplitude 2.5
Triangle Background with Spin 
waveform triangle background spin 500 rate 5.5 intensity 30 amplitude 10

Presets

Presets are reusable configurations that define a specific state or sound combination.

Syntax
[preset name]
[elements] ...

Example

Alpha Preset 
alpha
  noise brown amplitude 40
  tone 100 binaural 8 amplitude 15

Templating Presets

Create reusable base templates that can be inherited and modified by other presets.

Declaring a Template

Template Declaration 
base-focus as template
  noise white amplitude 25
  tone 180 binaural 8 amplitude 10

Creating Derived Presets

Derived Presets 
focus-phase1 from base-focus
  track 1 amplitude 20
  track 2 tone 200

focus-phase2 from base-focus
  track 1 amplitude 30
  track 2 tone 220
Note: Template presets cannot be used directly in the timeline. They serve as a base for derived presets.

Accepted Track Parameters

  • tone <value> - carrier frequency (Hz)
  • binaural <value> / monaural <value> / isochronic <value> - beat frequency (Hz)
  • amplitude <value> - loudness (0–100%)
  • intensity <value> - effect intensity (for spin/pulse)
  • spin <value> - spin width effect for background track
  • pulse <value> - pulse effect for background track

Timeline

The timeline controls when presets play and how they transition.

Syntax
hh:mm:ss [preset name]

Example

Timeline Example 
00:00:00 silence
00:00:15 alpha1
00:01:00 alpha1
00:02:00 alpha2
00:03:00 silence

Fade In/Out

Use the silence keyword to create smooth fade-ins and fade-outs.

Transitions Between Different Types

In SynapSeq V3+, transitions between presets are smooth for elements of the same type. For different types (e.g., binaural to isochronic), you must explicitly define the transition.

Method 1: Using Silence

Fade Between Different Types 
00:00:00 silence
00:00:15 ps1
00:02:00 ps1
# Fade-out
00:02:30 silence
# Fade-in
00:03:00 ps2

Method 2: Manual Amplitude Control

Amplitude Crossfade 
alpha1
  tone 300 binaural 10 amplitude 10
  tone 300 isochronic 10 amplitude 0

alpha2
  tone 300 binaural 10 amplitude 0
  tone 300 isochronic 10 amplitude 10

Transitions

Control how parameters change between presets over time.

Syntax
hh:mm:ss [preset name] [transition type]

Transition Types

steady Steady (Default)

Constant rate of change throughout the entire transition period.

Progress: 0% —— 25% —— 50% —— 75% —— 100% (uniform change)
Transition wave:

Characteristics:

  • Uniform progression from start to finish
  • Neutral, mechanical feel
  • Predictable and consistent

Best for: Testing, debugging, predictable linear changes

Provides a steady, predictable stimulus that works well as a baseline reference, though it may be more noticeable to the brain than natural transitions.

ease-out Ease-Out (Logarithmic)

Starts with rapid change and gradually slows down, creating a smooth landing.

Progress: 0% —— 60% —— 80% —— 90% —— 100% (fast start, gentle ending)
Transition wave:
0% 15% 35% 60% 85% 100%

Characteristics:

  • Most change happens early in the transition
  • Gradually decelerates as it approaches the target
  • Like a car smoothly braking to a stop

Best for: High to low frequencies (beta → theta), relaxation, meditation entry

Mimics the natural process of falling asleep or entering relaxation. The rapid initial change captures the brain's attention and begins the shift, while the gentle ending allows the nervous system to stabilize comfortably in the new state without resistance.

ease-in Ease-In (Exponential)

Starts gently and accelerates toward the end, creating a smooth departure.

Progress: 0% —— 10% —— 20% —— 40% —— 100% (gentle start, fast ending)
Transition wave:
0% 15% 40% 65% 85% 100%

Characteristics:

  • Starts slowly and gradually accelerates
  • Most change happens near the end
  • Like a car smoothly accelerating from a stop

Best for: Low to high frequencies (theta → beta), awakening, activation

Mirrors the natural awakening process. The gentle start avoids shocking the nervous system when emerging from deep states, while the accelerating finish firmly establishes the new alert state without leaving residual drowsiness.

smooth Smooth (Sigmoid)

Most natural feeling with gentle starts and endings, faster change in the middle.

Progress: 0% —— 20% —— 50% —— 80% —— 100% (slow → fast → slow, S-shaped)
Transition wave:
0% 5% 25% 50% 75% 95% 100%

Characteristics:

  • Starts slowly (ease-in)
  • Accelerates in the middle
  • Ends slowly (ease-out)
  • S-shaped curve, most natural and organic

Best for: General-purpose transitions, maximum comfort, therapeutic sessions

Provides the most comfortable and natural-feeling transition. The gentle start and end minimize perception of change, while the middle acceleration ensures the transition completes smoothly. This approach mimics natural processes and feels organic to the nervous system, making it ideal for therapeutic and meditative applications.

Example

Using Transitions 
00:00:00 silence
00:00:30 awake
# Entering meditation - use EASE-OUT
00:01:00 awake ease-out
00:06:00 deep
# Awakening - use EASE-IN
00:10:30 deep ease-in
00:15:00 awake
00:16:00 silence

Comments

Comments help document your sequences and are ignored during processing.

Syntax
# [comment]

Regular Comments

Single # Comment 
# This comment is ignored
alpha
  tone 440 binaural 8 amplitude 5

Printed Comments

Use double ## to print comments in the output.

Double ## Comment 
## This comment will be printed in the output
alpha
  tone 440 binaural 8 amplitude 5
Comments must occupy an entire line. Inline comments will cause syntax errors.

Global Options

Global options are set at the top of your file and start with @.

@background

Set a background audio file to loop throughout your sequence.

Examples 
@background /path/to/background.wav
@background ~/Downloads/rain.wav
@background https://example.com/sounds/rain.wav
Requirements:
  • Format: .wav files (8-bit/16-bit/24-bit)
  • Channels: 2 (stereo)
  • Sample rate must match sequence sample rate
  • Maximum file size: 10 MB

@gainlevel

Set the attenuation level of the background sound.

Example 
@gainlevel medium
off No attenuation (0dB) - full volume
high Light attenuation (-3dB)
medium Moderate attenuation (-9dB)
low Strong attenuation (-18dB)

@volume

Set the overall output volume (0-100).

Example 
@volume 75

Default: 100

@samplerate

Set the sample rate of the output.

Example 
@samplerate 44100

Default: 44100

@presetlist

Import presets from external files for reusability.

Examples 
@presetlist /path/to/my-presets.spsq
@presetlist ~/sequences/relaxation-presets.spsq
@presetlist https://example.com/presets/focus-presets.spsq
Only preset definitions are imported. Timeline sections and global options in preset files will trigger errors.

Command Line

Syntax
synapseq [options] [input file] [output file]

Basic Usage

Generate WAV from sequence 
synapseq sample-binaural.spsq sample-binaural.wav

Using stdin

You can use - to open sequence from stdin:

Pipe from stdin 
cat example.spsq | synapseq - output.wav

Using URLs

You can also use HTTP or HTTPS URLs to load sequences directly from the web:

Load from URL 
synapseq https://example.com/sequences/my-sequence.spsq output.wav

RAW Output Format

On *nix systems, you can play a sequence in RAW format using other audio tools:

Pipe to play (sox) 
synapseq sample-binaural.spsq - | play -t raw -r 44100 -e signed-integer -b 16 -c 2 -

RAW Audio Parameters

If you want to use another tool to process the output, keep in mind that the audio is emitted in RAW format with the following parameters:

  • Type: RAW
  • Sample Rate: 44100 Hz (default, configurable with @samplerate)
  • Encoding: Signed Integer
  • Bit Depth: 16
  • Channels: 2 (stereo)

Hub Commands

SynapSeq Hub is a centralized repository of community-contributed sequences. You can browse, search, download, and generate audio files directly from the Hub.

Explore sequences at SynapSeq Hub Repository.

Important: Before using any Hub command, you must first run -hub-update to fetch the latest manifest.

Privacy & Analytics

When you download sequences using -hub-get, -hub-info, or -hub-download, SynapSeq sends anonymous usage statistics.

What is collected:

  • Sequence ID (which sequence was downloaded)
  • SynapSeq version
  • Platform (Windows, macOS, Linux)
  • Architecture (amd64, arm64)
  • Action performed (get, info, or download)

What is NOT collected:

  • No personal information (names, emails, usernames)
  • No IP addresses or location data
  • No file paths or local directory structure
  • No browsing history or system information

The tracking is done asynchronously and never blocks or delays your downloads. All tracking logic is open source and can be fully inspected.

Compiling without Hub functionality: You can compile a Hub-disabled version using the nohub build tag. See the Compilation Guide for instructions.

-hub-update

Updates the local index of available sequences from the Hub.

Update Hub manifest 
synapseq -hub-update

-hub-list

Lists all available sequences in the Hub.

List all sequences 
synapseq -hub-list

-hub-search

Searches for sequences in the Hub by keyword. The search is case-insensitive and matches against the sequence ID, name, author, and category.

Search for sequences 
synapseq -hub-search focus

-hub-download

Downloads a sequence and all its dependencies to a local directory.

Download sequence 
synapseq -hub-download synapseq.samples.genesis ./downloads

If no target directory is specified, the current directory (.) is used.

-hub-get

Generates audio output directly from a Hub sequence without downloading it first.

Basic Usage (WAV output)

Generate WAV from Hub 
synapseq -hub-get synapseq.samples.genesis output.wav

Stream to stdout

Stream RAW audio 
synapseq -hub-get synapseq.samples.genesis -

Play in real-time

Stream and play 
synapseq -hub-get synapseq.samples.genesis -play

Convert to MP3

Generate MP3 from Hub 
synapseq -hub-get synapseq.samples.genesis -mp3

-hub-clean

Cleans the local Hub cache, removing all cached manifest data and downloaded temporary files.

Clean Hub cache 
synapseq -hub-clean

Note: After cleaning the cache, you'll need to run -hub-update again.

Additional Options

-extract

Extracts the original sequence and metadata embedded in a WAV or MP3 file.

Extract from WAV

Extract from WAV 
synapseq -extract input.wav output.spsq

Extract from MP3

To extract sequences from MP3 files, you must use the -mp3 flag and have ffprobe installed.

Extract from MP3 
synapseq -extract -mp3 input.mp3 output.spsq

-play

Plays a sequence in real-time using ffplay. The audio is streamed directly without creating intermediate files.

Requirements: This feature requires ffplay (part of the FFmpeg suite) to be installed.
Play sequence 
synapseq -play sample-binaural.spsq
Play from URL 
synapseq -play https://example.com/sequences/my-sequence.spsq

-mp3

Converts a sequence to MP3 format using ffmpeg. By default, creates an output file with the same name as the input but with .mp3 extension.

Requirements: This feature requires ffmpeg to be installed.
Auto-generate filename 
synapseq -mp3 sample.spsq
Custom output filename 
synapseq -mp3 sample.spsq output.mp3

MP3 Audio Quality

  • Codec: libmp3lame
  • Bitrate: 320 kbps (high quality)
  • Sample rate: Matches the sequence (default 44100 Hz)
  • Channels: Stereo (2 channels)

-convert

Converts a structured format (JSON, XML, YAML) back into .spsq format.

Convert JSON to SPSQ 
synapseq -convert -json input.json output.spsq

-json

Parse the input file as JSON format.

Process JSON sequence 
synapseq -json sequence.json output.wav

-xml

Parse the input file as XML format.

Process XML sequence 
synapseq -xml sequence.xml output.wav

-yaml

Parse the input file as YAML format.

Process YAML sequence 
synapseq -yaml sequence.yaml output.wav

-ffmpeg-path

Specifies a custom path to the ffmpeg executable.

Custom ffmpeg path 
synapseq -mp3 -ffmpeg-path /usr/local/bin/ffmpeg sample.spsq output.mp3

-ffplay-path

Specifies a custom path to the ffplay executable.

Custom ffplay path 
synapseq -play -ffplay-path /usr/local/bin/ffplay sample.spsq

-ffprobe-path

Specifies a custom path to the ffprobe executable. Only needed when extracting sequences from MP3 files.

Custom ffprobe path 
synapseq -extract -mp3 -ffprobe-path /usr/local/bin/ffprobe input.mp3 output.spsq

-help

Show the help and exit.

-unsafe-no-metadata

Prevents embedding the original SPSQ sequence and metadata into the generated WAV or MP3 file.

Generate without metadata 
synapseq -unsafe-no-metadata sample.spsq output.wav

-quiet

Quiet mode. Used to hide terminal output. Errors and comments will be displayed.

-test

Test mode. Used to validate the syntax of a .spsq file or structured formats (JSON, XML, YAML) without generating any output file.

Test sequence syntax 
synapseq -test sample-binaural.spsq

-version

Show the version.

-install-file-association (Windows only)

Installs Windows file association and context menu integration for SynapSeq.

Features:

File Association:

  • Double-click any .spsq file to automatically play the sequence
  • Assigns the SynapSeq icon to .spsq files in Windows Explorer

Context Menu for .spsq files:

  • "SynapSeq: Edit sequence" - Opens in Notepad
  • "SynapSeq: Test sequence" - Validates syntax
  • "SynapSeq: Convert to WAV" - Generates WAV file
  • "SynapSeq: Convert to MP3" - Generates MP3 file

Context Menu for .wav/.mp3 files:

  • "SynapSeq: Extract sequence" - Extracts embedded .spsq
Install file association 
synapseq -install-file-association

Note: This command modifies the Windows registry (HKEY_CURRENT_USER only). Administrator privileges are not required.

-uninstall-file-association (Windows only)

Removes all Windows file associations and context menu entries created by SynapSeq.

Uninstall file association 
synapseq -uninstall-file-association

Installing FFmpeg

To use the -play, -mp3, and -extract -mp3 features, you need to have FFmpeg installed on your system.

Download FFmpeg: https://ffmpeg.org/download.html

Installation by platform:

macOS (using Homebrew):

Install on macOS 
brew install ffmpeg

Linux (Ubuntu/Debian):

Install on Ubuntu/Debian 
sudo apt update
sudo apt install ffmpeg

Linux (Fedora):

Install on Fedora 
sudo dnf install ffmpeg

Windows (using winget):

Install on Windows 
winget update
winget install ffmpeg

Verify installation:

Check FFmpeg installation 
ffmpeg -version
ffplay -version
ffprobe -version

If these commands return version information, FFmpeg is correctly installed and available in your PATH.

Notes

File Size Limits

When using HTTP/HTTPS URLs for @background or @presetlist, there is a maximum file size limit of 10 MB.

Background Looping

SynapSeq automatically creates a seamless looping effect for background sounds. Ensure your audio file is loop-friendly for the best results.
